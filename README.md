# 🎮 Guess Game (Jogo de Adivinhação)

Um jogo de adivinhação de números baseado em console desenvolvido em Java, onde vários jogadores podem competir simultaneamente para adivinhar o número secreto gerado pelo sistema.

O projeto foi reestruturado e migrado para o padrão **Maven**, aplicando conceitos avançados de Engenharia de Software e Clean Code.

## 🚀 Tecnologias Utilizadas

* **Java 21**
* **Maven** (Gerenciamento de dependências e build automatizado)
* **Git & GitHub** (Controle de versão e chaves SSH para segurança)

## 🏗️ Arquitetura e Boas Práticas (Clean Code)

O projeto foi totalmente refatorado saindo de um script único para uma arquitetura inspirada no padrão **MVC (Model-View-Controller)**, dividindo as responsabilidades de forma isolada:

* **Model (Dados):** Classe `Player`, funcionando como um cofre encapsulado (atributos `private` com Getters e Setters) para proteger as informações dos jogadores.
* **View (Interface):** Classe `TelaDoTerminal`, um componente totalmente *Stateless* (sem estado), responsável estritamente pela interação com o usuário (inputs/outputs). Garante o tratamento do buffer do `Scanner` usando `Integer.parseInt(teclado.nextLine())`.
* **Controller (Regra de Negócio):** Classes `GuessGame` (O Maestro que coordena o fluxo) e `GameMath` (A engine matemática que dita as regras do jogo).

### 💡 Funcionalidades Destaque
* **Tratamento de Empates:** O sistema gerencia listas dinâmicas de vencedores (`List<String>`) para permitir que mais de um jogador vença na mesma rodada, listando todos de forma formatada.
* **UX no Terminal:** Efeito de suspense limpando a tela do console antes de anunciar os ganhadores.

## 📦 Como Compilar e Rodar o Jogo

Como o projeto agora é gerenciado pelo Maven, você pode buildar e rodar o projeto diretamente pelo terminal sem depender de uma IDE (Eclipse/IntelliJ).

### Pré-requisitos
* JDK 17+ instalado
* Maven instalado

# 🎮 Guess Game (Number Guessing Game)

A console-based number guessing game developed in Java, where multiple players can compete simultaneously to guess the secret number generated by the system.

The project was restructured and migrated to the **Maven** standard, applying advanced Software Engineering and Clean Code concepts.

## 🚀 Technologies Used

* **Java 17** (or the version you used)
* **Maven** (Dependency management and automated build)
* **Git & GitHub** (Version control and SSH keys for security)

## 🏗️ Architecture and Best Practices (Clean Code)

The project was completely refactored from a single script to an architecture inspired by the **MVC (Model-View-Controller)** pattern, cleanly separating responsibilities:

* **Model (Data):** `Player` class, acting as an encapsulated vault (`private` attributes with Getters and Setters) to protect player information.
* **View (Interface):** `TelaDoTerminal` class, a completely *Stateless* component, strictly responsible for user interaction (inputs/outputs). Ensures `Scanner` buffer handling using `Integer.parseInt(teclado.nextLine())`.
* **Controller (Business Logic):** `GuessGame` class (The Maestro coordinating the flow) and `GameMath` (The mathematical engine dictating the game rules).

### 💡 Highlight Features
* **Tie Handling:** The system manages dynamic lists of winners (`List<String>`) to allow more than one player to win in the same round, listing all of them cleanly.
* **Terminal UX:** Suspense effect by clearing the console screen before announcing the winners.

## 📦 How to Compile and Run the Game

Since the project is now managed by Maven, you can build and run it directly from the terminal without relying on an IDE (Eclipse/IntelliJ).

### Prerequisites
* JDK 17+ installed
* Maven installed
